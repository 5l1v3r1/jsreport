This first example should explain the most basic techniques used in jsreport.

Assembling reports in jsreport is based on javascript templating engines. You define a xml template like this one with special tags and jsreport will render report output for you. Output can be html, pdf and others will come. This example is rendering simple html, but you will learn about other output formats in the next examples.

In the integrated templated designer is the template on the left side. You can find a preview of the final report on the right side. Using "Run" butotn from the toolbar will invoke post to the jsreport server and displays the output in the previe. Go ahead and try it!

Every jsrender template contains a html and javascript helpers. This comes from basic concepts of javascript templating engines. The html contains the report design and the helpers can contain any javascript function that can do the some computations required in the report.
